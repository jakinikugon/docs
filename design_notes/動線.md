# 動線

## 献立提案の動線

### シナリオA: 献立が決まっている消費者

1. 今日の夕飯どうしよう
2. アプリを開く
3. いま冷蔵庫にあるものから、足りない食材を教えてくれる
4. 不足食材のうち、近隣店で割安な食材を提示教えてくれる（チラシみたいな）
5. お店に行き、購入をする
6. アプリ内の購入報告をすると自動で救済数が1増える

## シナリオB: 献立が決まっていない消費者

1. 今日の夕飯どうしよう
2. アプリを開く
3. 次の 3 つを全て提示する
   1. 冷蔵庫にあるものだけで作れるもの
   2. 近隣店で割安な食材を増やせば作れる料理
   3. 近隣店で割安な買うだけで食べれる料理や、お惣菜、お菓子
4. お店に行き、購入をする
5. アプリ内の購入報告をすると自動で救済数が1増える

## 余剰食材マッチングの動線

### シナリオC: 企業

1. 賞味期限が切れそう/ロスになりそうな食品がある、もったいないな
2. 出品に必要な情報を入れる
3. 出品に任意情報を与える
4. ボタンを押して出品完了
   - 在庫管理はしない
   - 必要なら
5. ユーザーが店頭で購入する
6. ユーザーがアプリ内の購入報告をすると自動で救済数が1増える

### シナリオD: 個人

1. 今日の夕飯どうしよう
2. アプリを開く
3. 検索機能で調べる
4. お店に行って購入する
5. アプリ内の購入報告をすると自動で救済数が1増える
