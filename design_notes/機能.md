# 機能

## 1. 冷蔵庫（家庭側）

- 最小実装: 手入力
  - 食材名
  - 分類（企業側の出品と対応）
  - フリーテキスト
- 最適実装: レシート読み込みで手選択
  - 頻出の食材サジェスト
  - 消費/消費期限
- 最上位実装: 冷蔵庫の画像から分類

### 実装しないこと

- 分量の入力

### その他設計

- よく使う食材のテンプレがあると入力しやすい
- 期限情報は最上位実装として扱う（入力負担が重いので）
- 冷蔵庫に食材を追加するときに、余剰食材**購入報告**も行う

## 2. 余剰食品出品（企業側）

- 入力時の情報が表示される、更新はされないことを明記

### 必須情報

- 商品名（任意の文字列）
- 通常価格（整数値）
- 消費/賞味期限（yyyy-mm-dd HH 頃）
- 写真（PNG/JPEG/HEIF/ほか）
- 種別を選択する（テキストのプルダウン）
  - 例: お肉/お魚/お乳製品/お野菜/お弁当/お惣菜/お菓子/その他（入力）

### 任意情報

- 割引後の価格（デフォルトは0円）
  - 時間遷移もできたらいいね
- 受け渡し場所（デフォルトはお店）
- アレルギー情報（デフォルトは不明）
- 数量（正の整数値）
  - 単位どうしようか問題
- 掲載期限（デフォルトは消費/賞味期限）
- その他詳細

### 実装しないこと

- 在庫管理
  - 掲載の手動取り消し・編集は可能だけど、基本的にしないことを想定

### その他設計

- 掲載経過時間表示
  - 残数保証なしの表示
- 入力時のタイムスタンプを自動取得
- CSV などで一括登録

## 3. 献立（レシピ）生成

- よねぴ UI 原案: ５
- レシピ: 作れる料理と材料名と余剰食品
- LLM に全部任せる
- ユーザーはチャットボットのようなもので献立を相談する
- タブで 献立決まってる/決まってないを分離
  - 既に献立が決まっている人に対しては、冷蔵庫にある食材の内、何が足りていないのか提示し、
    もし足りていない食材のなかで近隣のスーパーが余剰食材として
    出品している場合はおすすめする。
  - まだ何を作るか決めていない人に対しては、冷蔵庫内の食材で完結する献立提案と、
    近隣のスーパーの余剰食品をいくつか購入すると作ることのできる献立を両方提案する。
    - 無理やり余剰食材を買わせるのではなく、ユーザーの判断にゆだねる方向
- あくまで材料を提案するだけで、レシピや分量はサポートしない方向性

## 4. 余剰食材検索

- よねぴ UI 原案: １
- 商品名検索（フリーテキスト）
- 絞り込み: 分類/距離/期限までの時間/割引後の価格/通常価格/割引率/アレルギー/お店の救済数
- メルカリのような感じ
- あくまでスーパーのチラシのような立ち位置で、オンライン購入や配送は考えていない
- このスーパーにこれがあるから安く手に入るよといった感じ
